tcp/ip 2주차 수업

연결 장치 

1. 리피터 - 여러 개의 lan을 연결

2. 허브 - 여러 개 포트를 가진 리피터로 볼 수 있음, 허브는 컴퓨터 여러 대를 서로 연결하는 장치이기도 하며, 리티퍼와 마찬가지로 전기신호를 증폭하는 기능을 가지고 있음

3. 브리지 (Bridge)
-lan과 lan을 연결 : 이점은 리피터랑 비슷하다.
-필터링(filtering)기능이 있다 : 리티터랑 다른점

4.라우터(router)
-라우터는 여러 lan들을 연결하며 어드레스에 따라 알맞은 연결 라인을 찾아준다.
-wan에서 연결 다리 역할과 길찾아 주기 역할을 한다.

-광케이블(내부용)

허브(HUB)
-하나의 세그먼트안에서 두개이상의 네트워크 디바이스들을 연결 해주는 기능을 제공하는 PHYSICAL LAYER 장비

공유형 Ethernet의 문제점 - 대역폭

1. 단말이 늘어날 때 마다 전체 대역폭이 나누어진다.
2. 각 단말이 할당되는 대역폭은 10mbps/단말 수
3. 단말이 늘어날 때 마다 Collision(충돌)의 발생이 늘어난다.


공유형 Ethernet의 문제점 - 해결점(Ethernet 세그멘트를 나누고 브릿지로 연결)

1.브릿지는 lan을 더 적은 공유형 세그먼트로 나눈다.
2.unicast 트래픽은 해당 세그먼트내에서만 전송이 됨.
3.브로드캐스트 트래픽은 모든 세그먼트로 전송이 된다.
4.브리지는 collision을 전송하지 않는다. -> collision 도메인을 나눈다.


브리지
-네트워크 세그먼트 간을 연결하거나 프레임을 전달 해주는 기능을 수행하는 data link layer장비
-허브보다 지능적인 동작
-들어오는 프레임을 분석한다.
-forword 나 discard(필터링)는 mac주소에 따라서 결정한다.
-세그먼트내에서만 서로 전송 경쟁

스위치와 브리지의 비교

- 스위치는 처리방식이 mac 어드레스를 사용하고, 하드웨어로 이루어지기 때문에, 소프트웨어적으로 프레임을 처리하는 브리지에 비해서 훨씬 고속화가 가능

- 스위치는 브리지에 비해 제공하는 포트수가 휠씬 많음, 멀티포트 브리지

- 스위치 종류 : lan스위치, atm 스위치, wan 스위치

- 스위치 - learning, forward, filtering

- 스위치- 하드웨어, 브릿지 - 소프트웨어


스위치의 동작
-스위치는 브리지 기술을 기반으로 하였으며, cam(content addressable memory)라 블리는 어드레스 테이블을 작성하고 관리

라우터 : 네트위크 계층 어드레스(ip어드레스등) 사용, 출발지에서 목적지까지 경로를 결정, 브로드 케스트 영역을 나누어주는 성질이 있다.

라우팅 - 출발지에서 목적지까지의 길을 찾아준다.
